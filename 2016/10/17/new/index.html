限定查询

限定查询就是为了控制数据行的显示，可以根据条件进行数据行的筛选操作。在执行的顺序之中，WEHERE的执行顺序为：

3 -> SELECT ...
1 -> FROM ...
2 -> WHERE ...

限定查询的符号:

  • 关系运算符:>,<,<=,>=,<>(!=)
  • 逻辑运算符:AND,OR,NOT
  • 范围运算:BETWEEN…AND
  • 谓词范围:IN,NOT IN
  • 空判断:IS NULL,IS NOT NULL
  • 模糊查询:LINK

关系运算符的限定查询操作:

>,<,<=,>=,<>(!=)

1、查询出基本工资高于1500的雇员信息

SELECT *
FROM emp
WHERE sal>1500;

2、查询SMITH的基本信息

SELECT *
FROM emp
WHERE ename='SMITH';

在Oracle数据库中所有的数据是要区分一个大小写的关系的

3、查询工资是5000雇员信息

SELECT *
FROM emp
WHERE sal=1500;

在Oracle之中，所有的运算符号可以不区分数据类型进行运算

4、查询职位不是销售人员的雇员编号、姓名、职位

SELECT empno,ename,job 
FROM emp 
WHERE job!='SALESMAN';

SELECT empno,ename,job 
FROM emp 
WHERE job<>'SALESMAN';

这些运算符只是对数据进行单一的条件的筛选，基本的就是一个列加上一个属性！

逻辑运算符限定查询操作

AND、OR、NOT

如果现在在多个条件要进行查询，那要进行一个逻辑运算操作条件连接。

  • 与连接 所给的条件都要满足
  • 或连接 所给的条件只需要若干个满足即可
  • 非 如果条件条真则为假，如果条件为假则为真

  • 1、查询工资在1500~3000之间的雇员所有信息

    SELECT *
    FROM emp
    WHERE sal>=1500 AND sal<=3000;

两个条件都必需要满足！

2、查询工资大于2000的或者职位是CLERK的雇员信息。

SELECT * 
FROM emp
WHERE sal>2000 OR job='CLERK';

给出的条件只需要一个满足

3、查询工资小于2000的雇员信息

可以用 sal<2000，但可以利用NOT作以下形式:

SELECT * 
FROM emp
WHERE NOT sal>2000;

范围查询

BETWEEN...AND

BETWEEN可以用于数字和字符串

1、查询出工资在1500到2000的雇员信息

不用BETWEEN…AND

SELECT *
FROM emp
WHERE sal>=1500 AND sal<=2000;

用BETWEEN…AND

SELECT * 
FROM emp 
WHERE sal BETWEEN 1500 AND 2000;

2、查询在1981年入职的雇员

SELECT * 
FROM emp 
WHERE hiredate BETWEEN '01-1月-1981' AND '31-12月-1981';

空判断

IS NULL、IS NOT NULL

空在数据库上表示为不确定的内容，空在数据列上不为0，空的判断不为使用运算符进行运算！只能使用IS NULL IS NOT NULL(NOT IS NULL)

1、查询所有领取了薪水的雇员

领取了薪水那就是comm不为空

SELECT * F
ROM emp 
WHERE comm IS NOT NULL;

将IS NULL 取反

SELECT * 
FROM emp 
WHERE NOT comm IS NULL;

IN 操作符

IN、NOT IN

IN操作符和BETWEEN…AND相似，只不过BETWEEN…AND是给一个很大的一个范围！

1、查询雇员编号为7369、7599、9999、7788的雇员信息

不用IN

SELECT * 
FROM emp 
WHERE empno=7369 OR empno=7599 OR empno=9999 OR empno=7788;

使用IN操作符

SELECT * 
FROM emp 
WHERE empno IN (7369,7599,9999,7788);

2、查询雇员编号不为7369、7599、9999、7788的雇员信息

SELECT * 
FROM emp 
WHERE empno NOT IN (7369,7599,9999,7788);

取反

SELECT * 
FROM emp 
WHERE NOT empno IN (7369,7599,9999,7788);

!在使用NOT IN 进行操作时候遇到了NULL 就不会有结果返回

3、使用IN操作中有NULL

SELECT * 
FROM emp 
WHERE empno IN (7369,7599,9999,NULL);

却没有影响！

4、使用NOT IN操作中有NULL

SELECT * 
FROM emp 
WHERE empno NOT IN (7369,7599,9999,NULL);

却发现没有结果的返回。WHERE子句的作用就是限定数据行的返回，简单粗暴就是说不要给我返回全部的数据行，如果在一个数据表上有几千万条数据记录，一次性返回将会引发程序的崩溃。NOT IN 就是不在这给出的限定值中，有数据行永远不会为空的，那返回的将是全部的数据！

模糊查询

  • “_”匹配任意的一位字符

  • “%”匹配任意的零位、一位、多位字符

1、查询雇员姓名以A开头的信息

SELECT *　
FROM emp 
WHERE ename LIKE 'A%';

2、查询雇员姓名第二位为A的雇员信息

SELECT *　
FROM emp 
WHERE ename LIKE '_A%';

总结

WHERE 子句是放在FROM子句后面执行的，WHERE子句是控制数据行的显示，而SELECT是控制数据列的显示！
